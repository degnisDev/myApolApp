Creación de CRUD para tablas base de datos proyecto Back-end

Criterio de realización: Comprensión de la estructura de formatos Java, Json y XML
Contenidos temáticos que se abordarán en la actividad: Integración del Back-end con el servidor. Instalación de Herramienta de desarrollo. Las funciones de un CRUD (Create, Read, Update, Delete) y MVC.
Descripción de la actividad: Para el desarrollo de esta actividad es indispensable haber leído comprensivamente el material que se encuentra en esta etapa.
Indicaciones básicas para el desarrollo de la actividad:
Basado en los conocimientos adquiridos en las lecturas y videos de aprendizaje, se debe crear el Back-end del sitio web creado en la actividad práctica de la etapa de contextualización (el sitio Front-end entregado).
Tenga en cuenta las siguientes especificaciones:
Crear el Back-end para el sistema de administración que fue diseñado para incorporar los datos de la base de datos para la consulta solicitada. Se debe tener en cuenta que, para el acceso al módulo de administración, es necesario contar con un acceso basado en usuario o correo y una contraseña.
También debe incluir un CRUD por cada una de las tablas más importantes y/o que tengan relación con las consultas del sitio Web que evidenciara el público.
Puede utilizar la herramienta ATOM o el IDE Visual Studio Code para la gestión de su código, pero se recomienda utilizar Apache NetBeans por facilidad en el proceso.
Recuerde utilizar buenas prácticas de Arquitectura de software y, si se desea utilizar un Framework, se puede realizar.
Debe implementar la solución basada totalmente en el diseño previo del Front-end, aunque se permiten algunos ajustes para implementar el lenguaje de expresiones y/o el código correspondiente en JAVA, PHP, C# o cualquier otro con el que se sienta cómodo.
Entregable:
Una vez tenga elaborado el proyecto, debe guardar el desarrollo en una carpeta con la siguiente convención: AA1_SuPrimerApellido_SuSegundoApellido_SusNombres.
Esta carpeta debe incluir todas las carpetas del patrón de arquitectura, los archivos tanto de código como imágenes y estilos a utilizar.
Debe estar un archivo con la copia de seguridad de la base de datos. Por último, un archivo readme.txt donde se explique todo lo relacionado con requerimientos de instalación y configuración para el aplicativo junto con el lenguaje utilizado y las herramientas de desarrollo para visualizarlo.
