Interfaces gráficas completas de la tienda virtual Android

Criterio de realización: Diseño, modelación y desarrollo de aplicaciones para plataformas nativas

Contenidos temáticos que se abordarán en la actividad:

Diseño de activities mediante el uso de diferentes componentes y layouts
Uso de configuraciones de estilo, drawables y resources (values)
Generación del APK (Android Package) para validación
Uso de herramientas de versionamiento con Git y GitHub
Descripción de la actividad:

Cordial saludo, estimados estudiantes;

Ha llegado la hora de la aplicación de los conceptos teórico-prácticos abordados hasta el momento, pero aplicados directamente a su propuesta de desarrollo. En esta ocasión tendrán la oportunidad de hacer realidad su propuesta, específicamente realizando el desarrollo del frontend de su aplicativo. Para ello, es necesario recordar que la construcción de activities, fragments, menús y demás layouts y recursos debe encontrarse completamente terminada para todos los roles establecidos en la Actividad 2.

Esta actividad consiste en diseñar y construir las interfaces gráficas de la tienda virtual diseñada en la Actividad 2. Las pantallas mínimas requeridas por rol son:

Administrador: Dashboard principal, gestión de usuarios (listado, creación, edición, eliminación), gestión de productos y reporte de ventas.
Vendedor: Panel de productos (CRUD completo), historial de pedidos recibidos y perfil de vendedor.
Comprador: Pantalla de inicio, catálogo de productos, detalle de producto, carrito de compras, pasarela de pagos y perfil de usuario.
Transversal (todos los roles): Splash screen, autenticación con biometría, menú de navegación (Drawer / Bottom Navigation) y recuperación de contraseña.  
Para el desarrollo de esta actividad, tenga en cuenta las siguientes indicaciones:

Planificación: antes de comenzar a programar, recuerde seguir la estructura definida en la Actividad 2 de manera estructruada clara y secuencial de las pantallas y navegación por rol.

Grabación del video: utilice una herramienta de grabación de su elección. Asegúrese de tener buen audio e imagen clara. El video debe explicar el código a nivel técnico.

Recuerden tener en cuenta las observaciones realizadas en la Actividad 2 como oportunidades de mejora y solicitudes de completitud que permitan robustecer el desarrollo planteado.

Entregable:

Documentación técnica (Actividad 2) con las correcciones solicitadas, coherente con el desarrollo implementado.
Archivo comprimido (.zip) con el código fuente completo del proyecto Android Studio.
Enlace al repositorio en GitHub (acceso público) para realizar las validaciones correspondientes.
Video explicativo del desarrollo de interfaces con explicación técnica del código XML, layouts, drawables, estilos y values (máximo 5 minutos).  
Para el diseño de las interfaces gráficas y cada uno de sus componentes, se deben utilizar colores y propiedades diferentes a los que vienen por defecto.
